## CVE-2025-55184 & CVE-2025-55183

就在我们刚修完 CVE-2025-55182 没几天，ESAP 威胁情报中心又发来了新的安全通告。

React Server Components 又出事了。这次是两个：

### CVE-2025-55184（高危）—— 拒绝服务

攻击者可以构造恶意的 RSC Flight Payload，在服务器反序列化阶段触发异常的资源占用，导致进程阻塞或服务不可用。

**无需认证即可利用。**

翻译成人话：有人可以用精心构造的请求把你的服务器卡死。不需要登录，不需要任何权限，直接打。

### CVE-2025-55183（中危）—— 源码泄露

在特定输入条件下，服务器可能错误返回其他 Server Function 的编译后源码。

这个听起来没那么吓人？让我解释一下——**你的服务端逻辑、业务代码、敏感信息，都可能被泄露出去。**

虽然不是 RCE，但对于有敏感逻辑的项目来说，这同样是灾难。

### 影响范围

React 19.0.0 – 19.2.1，以及基于此的 Next.js App Router。

是的，我们上周刚升级到的 19.2.1，也在影响范围内。

真是「刚修好的盾，转头就被凿了两个洞」。

## 甲

从 1.0.0 到 1.2.2：**光 → 瞳 → 迹 → 网 → 炼 → 标 → 隐 → 韵 → 盾 → 甲**

上周我写「盾」的时候说，安全更新存在的意义是让你永远不需要知道它们有多重要。

没想到这么快就要写续集。

盾是被动防御，甲是主动护甲。盾可以丢掉，甲穿在身上。

**这就是为什么我们要持续更新——不是因为我们脆弱，而是因为威胁在进化。**

你可能会问：React 这个月到底怎么了？

我不知道。也许是安全研究员们突然对 RSC 产生了浓厚兴趣。也许是之前的漏洞引发了更多审计。也许只是巧合。

但这不重要。重要的是——

**漏洞会被发现，补丁会被发布，我们会更新。**

这是一个循环。不是完美的循环，但它在运转。

> **『没有永远安全的代码，』**
>
> **『只有永远警觉的维护者。』**
