## 格式修正

1.4.0 发布后，我发现 BEU 组织数据文件里有个问题——用了 markdown 格式标记（`**粗体**`），但组件不解析，直接显示原始文本。

页面上出现了一堆 `**系统级的异常干预模块**` 这种东西。

很丑。

所以我把三个语言版本的数据文件都清理了一遍：

- `data/organizations/zh-CN/beu.json` - 移除 5 处 markdown 标记
- `data/organizations/en/beu.json` - 移除 5 处 markdown 标记
- `data/organizations/ja/beu.json` - 移除 5 处 markdown 标记

现在文本就是文本，干净利落。

## 为什么不解析 markdown？

你可能会问：为什么不在组件里加 markdown 解析？

因为不需要。

数据文件应该保持纯粹。如果以后真的需要富文本，我会用更结构化的方式（比如 content blocks），而不是在字符串里塞 markdown。

**简单的问题，用简单的方案。**
